Upversioning

When a new version of a CRF is published and applied to a site, it does not automatically update existing CRFs. You can update those CRFs manually from the Upversion page. This page lists all records where the current revision number does not match the published revision number for a CRF. You can access Upversion from the left menu on the EDC.

You can filter the records by Site, Participant, Visit, CRF, Status, Record Version, and Record Revision using the filters in the white bar below the header.

Upversion field description

The following table outlines each column in the Upversion page, along with its description and purpose.

ColumnDescription
ParticipantShows the participant associated with the CRF.
VisitShows the visit associated with the CRF (for visit-based CRFs).
CRF NameShows the name of the CRF.
CurrentShows the current version and revision of the CRF associated with the visit.
PublishedShows the published version and revision of the CRF that the EDC will upversion the current CRF to.
CRF StatusShows the progress of the Current CRF, with statuses like:To DoIn ProgressCleaningTo ReviewDone

The following table details each CRF status, including its description and purpose.

CRF StatusDescription
To DoCRFs generated from visit group assignments during registration or randomization.
In ProgressCRFs where data entry has begun but not submitted.
CleaningCRFs with open queries, either manual or automatic.
To ReviewCRFs without open queries but pending tasks like review or verification.
DoneCRFs with no pending tasks or open queries.

Upversioning Records​

The following steps outline how to upversion records.

Steps to upversion records:​

  1. Use the available filters to narrow down the list of CRFs.
  2. Select the record(s) you want to upversion.

  1. In the top-right corner of the page, click UPVERSION.

A confirmation message appears, showing the number of records you selected.

  1. Click OK to proceed. The system upversions the selected records to the published revision and updates the build to the current version.
  2. After upversioning is complete, the records no longer appear on the Upversion page.
  • CRFs with statuses ‘In Progress’, ‘To Review’, ‘Cleaning’, or ‘Done’ change to ‘In Progress’ after upversioning.
  • CRFs with the ‘To Do’ status remain in ‘To Do’ status.
