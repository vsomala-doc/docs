Downloading Data

Details

Summary dropdown content

You can download and extract study data for analysis or record-keeping. Depending on your needs, you can download data at either the CRF level or the clinical domain level. The generated files can include additional details, such as audit history or metadata, based on the selected filters. You can find both CRF Data Download and Clinical Data Download in the left menu of the EDC dashboard.

teseting






Downloading CRF data​

Extract CRF-level data for a specific site, participant, or form. This is useful for reviewing or managing CRF data with the option to include audit history.

Steps to download CRF data​
  1. From the EDC dashboard, select CRF Data Download. The CRF Data Download page appears.

  1. Select the filter criteria: Site(s), Participant(s), CRF(s), and optional date ranges.
  2. Check Include audit history, Include PHI, or Include record metadata, if required.
  3. Select GENERATE. The generated file appears at the top of the ‘Previous Downloads’ table.

NOTE:

  • Use date filters to limit the data range.
  • Include PHI only when necessary and do not share with unauthorized users.

, Download Criteria field descriptions

The following table details each field in the ‘Download Criteria’ section, including its description and purpose.

FieldDescription
Select CRF(s)(Required)Select one, multiple, or all forms to download CRF-specific data.
Select Site(s)(Required)Select one or more sites to include data from specific locations.
Include PHIInclude protected health information (PHI) in the file.NOTE: To see this option, make sure you have the 'View PHI' permission.
Select Participant(s) (Required)Choose one, multiple, or all participants in the study.
Include record metadataCheck this box to include metadata for CRF.NOTE: File metadata is included in the record only when metadata is included in upload queries.
Include audit historyCheck this box to include an audit trail for CRF data changes.
Last Modified End Date(Optional) Filter data entered or modified before the specified date.
Last Modified Begin Date(Optional) Filter data entered or modified after the specified date.

Downloading clinical data​

Extract data at the domain level for broader analysis. This is useful for reviewing all clinical data across selected domains and participants.

Steps to download clinical data​
  1. From the EDC dashboard, select Clinical Data Download. The Clinical Data Download page appears.

  1. Select the filter criteria: Site(s), Participant(s), Domain(s), and optional date ranges.

  2. Check Exclude Labels if required.

  3. Select GENERATE. The generated file appears at the top of the ‘Previous Downloads’ table.

Download Criteria field descriptions

The following table details each field in the ‘Download Criteria’ section, including its description and purpose.

FieldDescription
Select Site(s)(Required)Select one or more sites to include data from specific locations.
Select Participant(s) (Required)Choose one, multiple, or all participants in the study.
Select Domain(s)(Required)Select one, multiple, or all domains to download domain-specific data.
Last Modified Begin Date(Optional) Filter data entered or modified after the specified date.
Last Modified End Date(Optional) Filter data entered or modified before the specified date.
Exclude LabelsCheck this box to exclude labels from the report.
Include PHIInclude protected health information (PHI) in the file.NOTE: To see this option, make sure you have the 'View PHI' permission.

Managing previously downloaded data​

The ‘Previous Downloads’ section lists all recent downloads for both CRF and clinical data. The table displays the status of your request and allows you to manage the downloaded files.

Statuses:

  • Processing: Indicates the file is being prepared. Select REFRESH to update the status.
  • Complete: The file is ready for download. A download icon will appear in the ‘Action’ column.

Actions:

  • Click the download icon to retrieve the file once it is ready.
