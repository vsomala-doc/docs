Managing Data Checks

The Data Checks page displays all active data checks defined for the studies in Study Builder. On the Data Checks page, you can run any data checks defined for the study across all sites or specific sites. You can track check status and view any queries the checks generate, update, or resolve on the Check Execution History page.

Select Data Checks from the left menu of the EDC to view data checks.

You can filter the data checks by Code, Category, and Domains using the filters in the white bar below the header.

Data Checks field description

The following table outlines each column in the Data Checks page, along with its description and purpose.

FieldDescription
CodeShows the code that identifies the data check.
VersionShows the version of the data check.
LabelShows the label corresponding to the data check.
CategoryShows the category associated with the data check.
DescriptionShows the text to the site when the data check issues queries.
PreviewShows the download icon to download data related to active data checks. The downloaded file will include clinical data from both the primary and secondary domains used in the data check.

Running Data Checks Manually​

For data checks with an active type that supports manual runs, an enabled checkbox appears to the right of the data check.

Steps to run data checks:​
  1. Look for data checks with an active type that allows manual runs.
  2. Select the checkbox next to each data check you want to run.
  3. Select one or more sites where you want to run the selected checks.

  1. Click RUN to execute the selected data checks for the selected sites. After you run data checks, EDC adds a new row to the Check Execution History page.

Checking Execution History​

After you run data checks, EDC adds a new row to the Check Execution History page, where you can view each run, including the checks, sites, and run status.

Steps to view data check execution history:​
  1. Click HISTORY on the Data Checks page.

The Check Execution History page appears.

  1. If required, click REFRESH to update the status.
  2. Filter the history by User, Code, Site, and Status using the filters in the white bar below the header.
  3. To view more details of a specific run, click a row to open the execution details for that data check run.

The Execution Details dialog appears with a row for each check code and site combination, along with its status and an overview of any queries generated.

  1. If the run was not completed successfully, a tooltip appears in the ‘Status’ column. Hover over or click the tooltip to view the error message.

Check Execution History field description

The following table outlines each column in the Check Execution History page, along with its description and purpose.

ColumnDescription
Date/TimeShows the timestamp when the data check started.
UserShows the user who started the data check.
Checks RunShows the name of the data checks that were run.
Site(s)Shows the site(s) where data check was run.
StatusShows the progress of the data check, with statuses like:To DoProcessingCompleteComplete with error

The following table details each data check status, including its description and purpose.

Data Check StatusDescription
To DoData check generated but has not started.
ProcessingData check has begun but has not completed yet.
CompleteData check completed successfully without errors.
Complete with errorData check completed with errors.
