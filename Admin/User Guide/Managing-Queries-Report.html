Managing Queries Report

You can use the queries report to view and manage all queries generated for a site, protocol, participant, CRF, or field. Queries, whether created automatically or manually, address potential data inconsistencies or observations.

Select Queries from the left menu of the EDC dashboard to open the report.

Query report field description

The following table outlines each field in the query report, along with its description and purpose.

FieldDescription
IDUnique identifier for each query, generated at the time of creation.
SiteShows the site associated with the query. The report can display queries from multiple sites.
PriorityIndicates the urgency of the query, categorized as Low, Medium, or High.
DescriptionProvides the reason for the query and any actions required, such as ‘Out of Range’ or ‘Missing Data’.
ParticipantShows the Participant ID associated with the query
VisitIndicates if the query is visit-related. If not, shows ‘N/A’.
CRFShows if the query is related to a CRF. If not, shows ‘N/A’.
QuestionDisplays the question label related to the query, if applicable.
StatusShows the query's status: Open, Answered, or Closed.
TypeIndicates the type of query: Manual, Automated, Out of Range, or Missing Data.
Check CodeShows the code that identifies the data check.
RoleLists the role that created the query. For system-generated queries, the role is ‘System’.
Last Updated DateShows the date of the most recent update to the query.

Filtering the query report​

You can filter the queries report to view specific records.

Steps to filter the query report:​

  1. On the Queries page, use the filters at the top to narrow the report by Site, Participant, Visit, CRF, Priority, Status, Type, Role, Active Queries, or Check Code.

  2. By default, only active queries are displayed. To view orphaned or deleted queries, update the filter options.

CRF-level and question-level queries​

CRF-level queries are raised to address potential data inconsistencies or observations at the Case Report Form (CRF) level. These queries help ensure data accuracy and compliance during clinical trials.

Question-level queries focus on specific data fields within a Case Report Form (CRF). These queries are generated to highlight potential inconsistencies, missing data, or other issues related to individual questions.

Viewing CRF-level and question-level queries​

You can view CRF-level and question-level queries from the queries report to check their status, description, and priority.

NOTE: Changes made to query status, priority, or answered reasons are automatically timestamped to maintain an accurate audit trail.

Steps to view and answer CRF-level and question-level queries:​

  1. From the EDC dashboard, select Queries. The Queries page appears.

  2. Use the filters at the top to narrow the report if required.

  3. Click the three horizontal dots on the far right of the query row and select Open [CRF Name] (e.g., Open Demographics).
    NOTE: CRF-level queries have ‘N/A’ in the ‘Question’ column.

NOTE: Question-level queries have a descriptive question in the ‘Question’ column.

  1. On the CRF page that opens, select the query icon to view the query details.

    NOTE: CRF-level query icons show at the CRF level.

NOTE: Question-level query icons show next to specific questions.

Edit query field descriptions

The following table details each field in the ‘Edit Query’ page, including its description and purpose.

FieldDescription
DescriptionDescribes the issue or concern related to the field or data.
StatusIndicates the status of the query: on hold, open, resolved, or closed.
PriorityIndicates the urgency of the query as low, medium, or high.
Created ByDisplays the role and the date when the query was created.
Opened ByShows the role and the date when the query was opened.
Answered Reason and DateProvides the reason and date for answering the query (visible for answered queries only).
Closed Reason and DateProvides the reason and date for closing the query (visible for closed queries only).

Query status icons​

EDC uses field icons to indicate the status of queries for a field. Each icon represents whether a query is open, answered, on hold, or closed. The following table explains these icons and their functionality based on user permissions:

Field IconDescription
Appears for users with ‘Add Query’ permission. Select this icon to create a new query for the field.
Appears for users with ‘View On Hold Queries’ permission. This icon indicates that there are queries in an ‘On Hold’ state for the field.
Appears for users with ‘View Open Queries’ permission. Indicates that the field has an ‘Open’ query.
Appears for users with ‘View Open Queries’ permission. Indicates that the query for the field has been ‘Answered’ but not yet closed.
Appears for users with ‘View Open Queries’ permission. Indicates that the query for the field has been ‘Closed’.

Adding comments to CRF-level and question-level queries​

You can add comments to CRF-level and question-level queries to provide additional context, clarify data inconsistencies, or document follow-up actions.

NOTE: While adding a comment to a query, you can also update the query status. If no status change is needed, select the current status to retain it.

Steps to add comments to CRF-level and question-level queries:​
  1. On the Edit Query page of the selected CRF, select NEW in the right corner of the ‘COMMENTS’ section.

The Enter Comment dialog box appears.

  1. Enter your comment, then select SAVE & UPDATE STATUS and select the status to save it or CANCEL to discard it. The comment is added to the ‘COMMENTS’ section.

Closing queries from the queries report​

You can close any query directly from the queries report to update its status, priority, or closure details.

NOTE: Changes made to query status, or priority details are automatically timestamped to maintain an accurate audit trail.

Steps to close queries from the queries report:​

  1. On the Queries page, select the applicable query row from the report. The Edit Query page appears.

  1. Select ‘Status’ from the dropdown list, choose ‘Closed’, and enter the close reason from the pre-defined list. Then select SAVE.

Adding a new query​

You can add a new query manually to address potential data inconsistencies or observations.

Steps to add a new query:​
  1. On the Queries page, select NEW in the top-right corner.

The New Query page appears.

  1. Select the filter criteria: Site, Participant, Visit, and CRF.

If the query is for non-participant, select the Non-Participant checkbox.

  1. Enter the reason for the query.
  2. Select the Status of the query.
  3. Select the Priority of the query.
  4. Click Save.

Adding comments to queries​

You can add comments to existing queries directly from the Queries page to provide context, clarify data inconsistencies, or document follow-up actions.

Steps to add comments:​

  1. Select the applicable query row from the report.

The Edit Query page appears.

  1. Follow the steps mentioned here to complete the process.
  2. Optionally, select AUDIT HISTORY to view detailed log of changes made to the query, including time of change, user, field type, old value, and new value.

Downloading the queries report​

You can download the queries report as a .xlsx file to your device. The report includes separate tabs for each section.

Steps to download the queries report:​

  1. On the Queries page, select DOWNLOAD in the top-right corner.

  2. After the download is complete, you will receive an alert. The report is saved to your device.

Uploading Queries​

Upload manual queries to existing CRFs using an Excel file. Download the template from the CRF Data Download page. Add queries with a Description, Status, and Priority, then import them from the Upload Queries page. Make sure you have 'Can Query' permission for the CRF and the relevant questions.

Steps to upload queries:​
  1. Download the template from Downloading CRF data with ‘Include record metadata’ selected.
  2. Add manual queries to the excel file. Make sure to include a Description, Status, and Priority for each query and save the file.
  3. On the Upload Queries page, click or drag to upload the Excel file.

  1. Select the Upload button located in upper-right corner of the ‘Upload Queries’ section.
  2. The excel file gets uploaded. The Uploaded file appears at the top of the ‘Previous Uploads’ section.

Previous upload field description​

The following table outlines each field in the ‘Previous Upload’ section, along with its description and purpose.

FieldDescription
Date/TimeShows time stamp of the file upload.
UserShows the Username who uploaded the file.
Source FileClick the Download icon to download the source file.
StatusShows the upload status of the file:If the file is currently uploading, the Status is set to Processing.Once the upload is complete, the Status is set to Complete.
Records UploadedIndicates the number of records imported within the file and records with errors.
ErrorsClick the Download icon to download a file that lists each failed record along with a description of the corresponding error.
